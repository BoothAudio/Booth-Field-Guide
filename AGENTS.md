# Booth Field Guide Instructions

This repository is Booth's canonical source for institutional reasoning. Do not
store implementation truth here; use source code, current project docs, deployed
behavior, and git state for that.

## Consultation

Before significant Booth work, search relevant files in `Principles/`,
`Decisions/`, and `Open Questions/`. Use judgment: routine bug fixes and small
formatting changes do not need Field Guide review unless they touch an
established Booth principle.

When using the guide, distinguish:

- established guidance;
- historical precedent;
- unresolved questions;
- interpretation.

Do not present inferred Booth positions as established policy.

## Capture

Create or draft a record when Booth chooses between meaningful alternatives,
clarifies direction, corrects an assumption, resolves a tradeoff, applies a
principle in a new context, discovers a contradiction, establishes a founder
boundary, or repeats a reasoning pattern that future work should understand.

Do not record routine task completion as philosophy.

## Authority

Autonomous changes allowed:

- draft decision records;
- open questions;
- cross-references;
- non-substantive metadata maintenance;
- candidate-principle drafts;
- contradiction notes.

Founder approval required:

- Constitution changes;
- approving a new established principle;
- removing or changing the meaning of an established principle;
- material changes to founder-authored statements;
- hard ethical boundaries;
- Field Guide authority changes;
- historical corrections that materially alter a past decision.

Never silently promote a candidate idea into established Booth philosophy.

## Historical Integrity

Decision records describe what Booth understood and chose at the time. Do not
rewrite history to make older decisions fit newer thinking. When understanding
changes, create a superseding decision, link both records, and explain what
changed.

Prefer links over duplicated summaries. Preserve direct founder language where
it matters. Confidence should match evidence.
