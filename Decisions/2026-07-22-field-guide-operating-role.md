# Decision: Make the Field Guide Booth's Canonical Reasoning Source

Status: Accepted
Date: 2026-07-22
Deciders: Nick

## Context

Booth's product, design, engineering, and company decisions were being preserved
across several places: workspace memory, daily notes, repo docs, source code,
and conversation history. Nick created a separate Booth Field Guide so future
Booth decisions can reference institutional reasoning without turning personal
agent memory into the canonical product philosophy.

## Founder Statements

Nick stated that operational memory helps the agent operate, project
documentation describes current systems and implementation, and the Booth Field
Guide records how Booth reasons, what Booth has learned, and why significant
decisions were made.

Nick also stated that the Field Guide repository must remain the canonical
source of Booth philosophy and institutional reasoning.

## Decision

The Booth Field Guide is Booth's canonical source for institutional reasoning:
constitutional values, established and candidate principles, significant
decisions, unresolved questions, and historical rationale.

Operational memory should retain only the minimum information needed to locate
the guide, understand what it is authoritative for, know when to consult it, and
know when to propose new records.

## Reasoning

This separates memory roles cleanly:

- source code and current docs remain implementation truth;
- operational memory helps the agent function across sessions;
- daily memory keeps raw activity context;
- the Field Guide preserves Booth reasoning and decision precedent.

The separation should reduce founder effort because the agent is responsible for
consulting and drafting records when meaningful Booth knowledge appears.

## Consequences

Before significant Booth work, the agent should consult relevant Field Guide
records. After meaningful decisions, the agent should draft appropriate records
without waiting for Nick to initiate documentation.

The agent may draft candidate principles and decision records, but founder
approval remains required for Constitution changes, established principles, hard
boundaries, authority changes, and meaning-changing historical edits.
